📩 Formato de Respuesta — Envelope Estándar

Todos los endpoints devuelven siempre la misma estructura envolvente (envelope pattern):

// ✅ Respuesta exitosa con lista (200 OK) { "success": true, "message": "Productos listados correctamente", "data": [ { ... }, { ... } ], // array de objetos "total": 5 // solo en listados } // ✅ Respuesta exitosa con objeto único (200 OK / 201 Created) { "success": true, "message": "Producto encontrado", "data": { "id": 1, "sku": "PRD-001", ... } } // ❌ Respuesta de error (404 / 422 / 500) { "success": false, "message": "Producto no encontrado: 99", "timestamp": "2024-01-15T10:30:00" } // ❌ Sin autenticación (401 Unauthorized) — respuesta de Spring Security { // No body, solo header WWW-Authenticate: Basic realm="Realm" }

💻 Ejemplos con cURL (para pruebas)

# Listar productos (usuario: admin, password: admin) curl -u admin:admin http://localhost:8080/api/productos # Crear un producto curl -u admin:admin -X POST http://localhost:8080/api/productos \ -H "Content-Type: application/json" \ -d '{"sku":"PRD-001","nombre":"Arroz Extra","categoriaId":1,"unidadId":1,"precioLista":3.50,"activo":true}' # Obtener producto por ID curl -u admin:admin http://localhost:8080/api/productos/1 # Actualizar producto curl -u admin:admin -X PUT http://localhost:8080/api/productos/1 \ -H "Content-Type: application/json" \ -d '{"nombre":"Arroz Extra Premium","precioLista":4.00}' # Eliminar producto curl -u admin:admin -X DELETE http://localhost:8080/api/productos/1

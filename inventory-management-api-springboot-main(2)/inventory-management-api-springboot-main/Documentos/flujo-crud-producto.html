⚠ GlobalExceptionHandler — Manejo centralizado de errores

La clase GlobalExceptionHandler.java (anotada con @RestControllerAdvice) intercepta todas las excepciones lanzadas desde cualquier RestController y las convierte en respuestas JSON consistentes. Sin él, Spring retornaría HTML de error.

@RestControllerAdvice public class GlobalExceptionHandler { // NotFoundException → 404 Not Found // Cuándo se lanza: producto/categoría/unidad no encontrado por ID @ExceptionHandler(NotFoundException.class) ResponseEntity<ErrorResponse> handleNotFound(NotFoundException ex) { return ResponseEntity.status(HttpStatus.NOT_FOUND) .body(ErrorResponse.of("NOT_FOUND", ex.getMessage())); } // BusinessException → 422 Unprocessable Entity // Cuándo se lanza: SKU duplicado, stock insuficiente para SALIDA, etc. @ExceptionHandler(BusinessException.class) ResponseEntity<ErrorResponse> handleBusiness(BusinessException ex) { return ResponseEntity.status(HttpStatus.UNPROCESSABLE_ENTITY) .body(ErrorResponse.of("BUSINESS_RULE", ex.getMessage())); } // Exception genérica → 500 Internal Server Error // Error inesperado: conexión DB perdida, NullPointer no manejado, etc. @ExceptionHandler(Exception.class) ResponseEntity<ErrorResponse> handleGeneric(Exception ex) { return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR) .body(ErrorResponse.of("INTERNAL_ERROR", "Ocurrió un error inesperado")); } }

📌 Tabla de Códigos HTTP del Sistema

Código Nombre Cuándo ocurre Quién lo genera
200 OK Success GET, PUT, DELETE exitoso RestController
201 Created Created POST exitoso (recurso creado) RestController
400 Bad Request Bad Request BusinessException (SKU duplicado, etc.) Service → Controller
401 Unauthorized Unauthorized Sin header Auth o credenciales incorrectas Spring Security
403 Forbidden Forbidden Autenticado pero sin rol ADMIN en endpoint restringido Spring Security
404 Not Found Not Found NotFoundException (ID no existe en DB) Service → GlobalExceptionHandler
422 Unprocessable Business Rule BusinessException manejada por GlobalExceptionHandler GlobalExceptionHandler
500 Internal Error Server Error Error inesperado (DB caída, NullPointer, etc.) GlobalExceptionHandler

🌟 Resumen: El ciclo completo de una petición

PETICIÓN COMPLETA: POST /api/productos (crear producto nuevo) Angular (Frontend) │ Authorization: Basic YWRtaW46YWRtaW4= ← base64("admin:admin") │ Body: { sku, nombre, categoriaId, ... } ▼ Spring Security — BasicAuthenticationFilter │ Decodifica Base64 → "admin:admin" │ UserDetailsServiceImpl.loadUserByUsername("admin") │ └─ SQL: SELECT * FROM Usuario WHERE usuario='admin' │ └─ new UsuarioPrincipal(usuario) ← dentro de @Transactional │ BCryptPasswordEncoder.matches("admin", "$2a$10$...") → true │ SecurityContextHolder.setAuthentication(token) ▼ ProductoRestController.crear(@RequestBody ProductoRequestDto) │ Bean Validation: @NotBlank, @Size, @DecimalMin, etc. │ productoService.crear(request) ▼ ProductoServiceImpl.crear(request) ← @Transactional │ sku = request.getSku().trim().toUpperCase() "prd-001" → "PRD-001" │ productoRepo.findBySku("PRD-001") → ¿ya existe? Si → BusinessException (400) │ categoriaRepo.existsById(2) → ¿existe? No → NotFoundException (404) │ unidadRepo.existsById(1) → ¿existe? No → NotFoundException (404) │ mapper.toEntity(request) → Producto entity │ entity.setCreadoEn(LocalDateTime.now()) │ productoRepo.save(entity) ▼ MySQL │ INSERT INTO producto (sku, nombre, precio_lista, ...) VALUES ('PRD-001', ...) │ AUTO_INCREMENT asigna producto_id = 1 │ Retorna entity con id=1 ▼ ProductoMapper.toResponseDto(saved) │ Producto entity → ProductoResponseDto ▼ ProductoRestController │ { success:true, message:"...", data: { id:1, sku:"PRD-001", ... } } │ ResponseEntity.status(201 CREATED).body(response) ▼ Angular (Frontend) │ Recibe { success: true, data: { id: 1, sku: "PRD-001", ... } } │ Actualiza la tabla de productos en pantalla

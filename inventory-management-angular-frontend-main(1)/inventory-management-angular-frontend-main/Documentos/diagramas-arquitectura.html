05 · Spring Security — Cómo protege el backend

Spring Security intercepta todas las peticiones HTTP antes de que lleguen al controlador. El sistema usa dos filter chains independientes.

Filter Chain 1: REST API (/api/**)
📥 Llega petición HTTP con header
Authorization: Basic YWRtaW46MTIzNA==
🔍 BasicAuthenticationFilter
Decodifica Base64 → "admin:1234"
🗄 UserDetailsService.loadUserByUsername("admin")
Busca usuario en tabla usuarios de MySQL
🔐 PasswordEncoder.matches()
Compara "1234" con BCrypt hash almacenado
✅ SecurityContext actualizado
Guarda el Authentication para la petición
🎯 Llega al RestController
Procesa GET/POST/PUT/DELETE
Filter Chain 2: MVC / Admin Web (/admin/**)
📥 Petición web (navegador al backend)
Sin header Authorization
🍪 UsernamePasswordAuthenticationFilter
Form login con sesión (cookie JSESSIONID)
🔐 Misma validación BCrypt
✅ Sesión HTTP establecida
🎯 Llega al Controller (Thymeleaf MVC)
⚠️ Esta cadena es para el backend web de administración, NO es usada por el frontend Angular.
🚫 ¿Qué pasa cuando Spring Security RECHAZA la petición?
Credenciales incorrectas o token ausente en la petición
→
Spring Security responde HTTP 401 Unauthorized
WWW-Authenticate: Basic realm="ERP" (el proxy lo elimina)
→
Angular: catchError en AuthService limpia localStorage y muestra el error al usuario

Diagrama 5: Spring Security actúa como portero antes de que cualquier dato llegue al controlador. El frontend Angular solo interactúa con la Chain 1 (REST).

06 · Ciclo CRUD Completo — Ejemplo Producto

Las cuatro operaciones fundamentales de la aplicación y el flujo exacto de cada una, usando el recurso Producto como caso de estudio.

C
CREATE
POST /api/productos
1. Clic "+ Nuevo Producto"
2. modal.set(true), form.reset()
3. Usuario llena formulario
4. guardando.set(true)
5. productoService.create(body)
6. Interceptor añade Basic Auth
7. Spring guarda en MySQL
8. HTTP 201 → next()
9. cargar(), alerta OK
10. modal.set(false)
R
READ
GET /api/productos
1. Router navega /productos
2. Angular instancia componente
3. ngOnInit → cargar()
4. cargando.set(true)
5. forkJoin(3 requests)
6. Interceptor + proxy
7. Spring Security valida
8. JPA SELECT productos
9. map(r => r.data)
10. *ngFor renderiza tabla
U
UPDATE
PUT /api/productos/{id}
1. Clic "Editar" en fila
2. modoEdicion.set(true)
3. idEditando = producto.id
4. form.patchValue(producto)
5. modal.set(true)
6. Usuario modifica campos
7. guardando.set(true)
8. productoService.update(id, body)
9. Spring UPDATE en MySQL
10. cargar(), alerta OK
D
DELETE
DELETE /api/productos/{id}
1. Clic "Eliminar" en fila
2. confirm('¿Seguro?')
3. Si cancela → nada
4. Si confirma → delete(id)
5. Interceptor + proxy
6. Spring Security valida
7. Verifica FK MySQL
8. HTTP 200 → next()
9. cargar(), alerta OK
⚠ HTTP 409 → alerta ERROR
